## The Testing Pyramid

- Unit tests: many, fast, focused
- Integration tests: fewer, broader
- End-to-end tests: fewest, slowest, highest environment cost

---

## What This Talk Focuses On

- Frontend unit tests
- Tight feedback while coding
- Small behaviors proved early

---

## Why This Level Matters

- You can run tests while you are still designing
- Failures point to one small behavior quickly
- The feedback loop stays short enough to be useful

---

## What That Means

- Not one big browser test
- Not full-system proof
- Clear checks for small frontend behavior

This is where TDD is easiest to learn well and easiest to apply consistently.
